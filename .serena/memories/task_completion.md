# Task Completion Checklist

## Verification Process
Before completing any task, perform these manual checks as there is no automated test framework:
- **Layout:** Verify both desktop and mobile layouts on edited pages.
- **Links:** Confirm all key links (Google Play, LINE, policy pages, back navigation) are functional.
- **Functionality:** Ensure animations and loaders run without console errors.
- **Media:** Validate that updated images load correctly and paths resolve from each HTML file.

## Pull Request Requirements
If preparing a PR, ensure it includes:
- **Description:** What changed and why.
- **Scope:** Affected pages and paths.
- **Evidence:** Before/after screenshots (desktop and mobile) for UI changes.
- **Testing:** Summary of manual test steps and results.
- **Context:** Linked issue or task if applicable.

## Commit Guidelines
- Use emoji-prefixed, concise commit messages.
- Ensure all new/updated badges follow the date format rule (`YYYY/MM/DD`).
- Optimize all new or updated images before committing.
