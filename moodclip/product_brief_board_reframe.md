# Product Brief: MoodClip Board Reframe

## Agent履歴

- Agent: Strategist

## 背景

- これまでの MoodClip は Web サイト保存前提のブックマークアプリとして整理されていた。
- ただし、プロダクトの本質的な強みは URL 管理ではなく、集めた素材を美しく見せる誌面的な体験にある。
- 今後は URL に限らず、画像、短文、音楽リンク、位置リンクなどをまとめて扱えるムードボードアプリへ再定義する。

## 目的

- MoodClip を、共有された素材を美しく並べて SNS 投稿用画像へ仕上げられるアプリとして再定義する。
- 保存対象の幅を広げても、今の「おしゃれでモダンで SNS 映えする見た目」は維持する。

## 対象ユーザー

- 気になったものを集めて世界観ごと残したいユーザー
- 情報整理よりも、雰囲気や見せ方を重視するユーザー
- SNS 投稿用に見栄えのよいまとめ画像を作りたいユーザー

## 利用シーン

- ブラウザ、SNS、音楽アプリ、地図アプリなどから素材を共有して保存する
- 保存した素材を 1つのボードに混在させて眺める
- できあがった見た目を 1枚の画像として書き出し、SNS に投稿する

## 解決したい課題

- URL だけでは世界観を作れず、保存対象が限定されすぎる
- 既存の「見せる強み」を活かすには、異種コンテンツを混ぜられる方が自然
- SNS 投稿用の見た目を整えたいのに、汎用メモやブックマークでは仕上がりが弱い

## MVP

- ボード / クリップの概念へ再定義する
- 共有入力として `URL / 画像 / テキスト / 音楽URL / 位置URL` を扱う
- 1つのボード内に異種クリップを混在表示できる
- テーマごとに複数の自動レイアウトプリセットを持つ
- 共有保存動作を設定で切り替えられる
  - 即時で未分類ボードへ保存
  - 保存時に保存先ボードを選ぶ
- 現在のボードを tall image 1枚として書き出せる

## Out of Scope

- 他ユーザーへのテンプレ共有
- 同期や共同編集
- 自由配置のコラージュ編集
- 深い音楽再生機能
- 深い地図操作機能
- 既存 bookmark データとの互換や移行

## UX方針

- 主役は常にボード表示
- UI は一覧管理より誌面感と空気感を優先する
- 共有は「投稿用画像の書き出し」を意味し、データ共有を意味しない
- 保存導線は Android 共有を中心に保つ

## 成功条件

- URL 以外の素材を受けても MoodClip らしい見た目が崩れない
- ボードが SNS 投稿向けの完成画として成立する
- 保存対象が増えても、保存導線が煩雑になりすぎない

## 制約

- Android を優先する
- 実装詳細は Sprint Contract で具体化する
- 見た目品質の後回しは禁止

## 不明点

- 音楽URL / 位置URL からどこまで補助メタデータを取得するかは実装スプリントで具体化する
- ボード別レイアウトプリセット UI の最終形は実装段階で詰める

## Plannerへの依頼

- まずは foundation reframe を実装契約に落とし込み、命名、設定、保存入口、表示基盤を再定義する
- 共有入力の全対応を一度にやり切らず、最初のスプリントは app-wide foundation を優先する
